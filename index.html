CH 1

1. Project Overview

A mobile e-commerce application developed using Flutter that allows users to browse products, manage favorites, update product information, and navigate through a modern and responsive user interface.

Project Objectives

  • Provide a simple shopping experience.
  • Practice Flutter application development.
  • Apply State Management using Provider.
  • Implement local data persistence.
  • Integrate external services such as Google Maps.

2. Project Idea

ShopEase is a shopping application that enables users to:

  • Create an account and log in.
  • Browse available products.
  • View product details.
  • Add products to favorites.
  • Add, update, and delete products.
  • Manage personal profile information.
  • Open Google Maps for navigation.

The application focuses on simplicity, usability, and efficient state management.

3. Technologies Used

Flutter Cross-platform framework used to build the application.

Dart Programming language used for development.

Provider Used for State Management and updating the UI dynamically.

Shared Preferences Used to save login status locally.

Google Maps Used for navigation and location services.

Material Design Used to create a modern and responsive user interface.

4. Application Flow

  • User logs into the application.
  • Products are displayed on the Home Screen.
  • User opens Product Details.
  • User can add products to Favorites.
  • User can manage products through CRUD operations.
  • User can access Profile and Maps features.

5. Authentication

The authentication module provides secure access to the application.

Features:

  • Login Screen.
  • Register Screen.
  • Input validation.
  • Persistent login using Shared Preferences.
  • Automatic session restoration.

Benefit:

Improves user experience by keeping users logged in between sessions.

6. Home Screen

The Home Screen is the central part of the application.

Features:

  • Product Grid Layout.
  • Responsive User Interface.
  • Easy Navigation.
  • Product Browsing Experience.

Technologies:

  • Flutter Widgets.
  • Provider State Management.

7. Product Details

Displays complete information about the selected product.

Features:

  • Product Image.
  • Product Name.
  • Product Description.
  • Price Information.
  • Favorite Option.

Benefit:

Allows users to view detailed product information before making decisions.

8. CRUD Operations

The application supports full CRUD functionality.

Benefits:

Provides complete product lifecycle management.

9. Favorites System

Allows users to save products they like.

Features:

  • Add to Favorites.
  • Remove from Favorites.
  • Instant UI Updates.
  • Shared State using Provider.

Benefits:

Provides quick access to preferred products.

10. Profile & Maps

Profile:

  • User Information.
  • Account Management.
  • Logout Functionality.

Google Maps:

  • Integration with Google Maps.
  • Navigation to predefined destination.
  • Easy access to location services.

11. State Management

Provider was used because it:

  • Reduces unnecessary widget rebuilding.
  • Simplifies data sharing.
  • Improves code organization.
  • Updates UI automatically when data changes.

Used In

  • Products List.
  • Favorites System.
  • Product Updates.

12. Conclusion

ShopEase demonstrates:

  • Flutter Development.
  • State Management using Provider.
  • Local Storage using Shared Preferences.
  • CRUD Operations.
  • Favorites Management.
  • Google Maps Integration.
  • Responsive User Interface.
